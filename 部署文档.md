# STUN 代理/重定向服务 - 部署文档

## 项目简介

STUN 穿透管理服务，支持三种模式转发到实际 STUN 服务端口。

**功能特性：**
- Webhook 自动创建/更新规则
- 域名代理/重定向：`前缀.端口.stun.sd8.cc` → 实际服务地址
- nginx 直通代理（proxy_mode=1）：S3/MinIO 等签名服务完美兼容
- 管理后台：用户管理、规则管理、系统设置
- RBAC 权限控制
- 防暴力破解（5次失败锁定15分钟）

**默认登录：** admin / admin123

---

## 〇、转发模式说明

| 模式 | 描述 | 适用场景 | S3 兼容 |
|------|------|----------|---------|
| **重定向** (proxy_mode=0) | 返回 307/308，客户端重定向到目标 | 普通 HTTP 服务，如 frp | ❌ 签名必挂 |
| **代理模式** (proxy_mode=1) | nginx 直通后端或 Python 代理转发 | 任何服务，包括 S3/MinIO | ✅ 完美（Host 保留） |

### 如何选择
- **一般 HTTP 服务**（frp, web 等）：用**重定向模式**（性能最好，浏览器自动跟随）
- **S3/MinIO 等签名服务**：用**代理模式**（nginx 直通可达 100+ req/s）

## 一、服务端部署（Python）

### 1. 上传文件
将 `stun_redirect_server.py` 上传到服务器 `/root/` 目录

### 2. 安装依赖
无需安装，纯 Python 标准库

### 3. 配置 Systemd 服务

创建 `/etc/systemd/system/stun-redirect.service`：

```ini
[Unit]
Description=STUN Redirect/Proxy Server
After=network.target

[Service]
Type=simple
ExecStart=/usr/bin/python3 /root/stun_redirect_server.py
Restart=always
RestartSec=5
WorkingDirectory=/root

[Install]
WantedBy=multi-user.target
```

```bash
systemctl daemon-reload
systemctl enable stun-redirect
systemctl start stun-redirect
```

### 4. 验证
```bash
# 检查服务状态
systemctl status stun-redirect

# 检查端口
netstat -tlnp | grep 8800

# 查看日志
journalctl -u stun-redirect -n 50 --no-pager

# 跟踪日志
journalctl -u stun-redirect -f
```

### 5. 停止/重启
```bash
systemctl stop stun-redirect
systemctl restart stun-redirect
```

## 二、宝塔 Nginx 配置

### 1. 添加站点
- 域名：`你的域名`（如 `stun.sd8.cc`）
- PHP 版本：纯静态

### 2. 添加反向代理
站点设置 → 反向代理 → 添加反向代理：
- 代理名称：`stun`
- 目标 URL：`http://127.0.0.1:8800`

### 3. 修改代理配置
**站点设置 → 配置文件**，找到 `location ^~ /` 块，修改为：

```nginx
location ^~ /
{
    proxy_pass http://127.0.0.1:8800;
    proxy_set_header Host $http_host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header REMOTE-HOST $remote_addr;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection $connection_upgrade;
    proxy_http_version 1.1;
    proxy_redirect off;
}
```

### 4. 修改 server_name
站点配置中 `server_name` 改为：

```nginx
server_name 你的域名 *.你的域名;
```

例如：
```nginx
server_name stun.sd8.cc *.stun.sd8.cc;
```

### 5. 配置 nginx 引入 stun.direct.conf（代理模式必须）

在 `/www/server/nginx/conf/nginx.conf` 的 `http {}` 块中（在 `include` 其他 conf 之前）添加：

```nginx
include /www/server/panel/vhost/nginx/stun.direct.conf;
```

这个文件由 `stun_redirect_server.py` 自动生成，包含 proxy_mode=1 规则的 nginx 直通 `server` 块，每个 `server_name` 精确匹配后直接 `proxy_pass` 到目标，**不经过 Python 层**，绕过 HEAD→GET 转换问题。

### 6. 重载 Nginx
```bash
nginx -t && nginx -s reload
```

## 三、架构说明

### 请求流程

```
客户端 → nginx (80/443)
         ├── stun.direct.conf 匹配 → 直通目标服务器（proxy_mode=1，不经过 Python）
         └── 站点代理 → Python :8800
                        ├── 域名重定向（proxy_mode=0 → 307/308）
                        └── 管理后台 / API
```

### 关键组件

| 组件 | 文件 | 说明 |
|------|------|------|
| 主程序 | `/root/stun_redirect_server.py` | 单文件，含嵌入 HTML/JS/CSS |
| Systemd 服务 | `/etc/systemd/system/stun-redirect.service` | Restart=always，崩溃自动恢复 |
| nginx 直通配置 | `/www/server/panel/vhost/nginx/stun.direct.conf` | 自动生成，proxy_mode=1 规则 |
| nginx 站点代理 | `/www/server/panel/vhost/nginx/proxy/stun.sd8.cc/*.conf` | 宝塔自动生成，代理到 :8800 |
| 数据库 | `/root/stun_redirect.db` | SQLite，WAL 模式 |

## 四、关键配置说明

### 必须正确的配置项

| 配置项 | 正确值 | 错误值 | 影响 |
|--------|--------|--------|------|
| `proxy_set_header Host` | `$http_host` | `$host` | 多级子域名前缀丢失 |
| `proxy_pass` | `http://127.0.0.1:8800` | `http://外部IP:8800` | 请求绕行导致 Host 被改 |
| `proxy_redirect` | `off` | 默认 | Location 头被 nginx 改写 |
| `server_name` | `域名 *.域名` | 正则写法 | 多级子域名匹配异常 |
| `proxy_pass` 中变量 | 避免使用 | `$stun_backend` | HEAD→GET 转换，签名失败 |

### 已知问题及修复

#### 1. HEAD→GET 转换（proxy_mode=1 用 stun.direct.conf 解决）
**现象：** S3 HEAD 请求返回 403，服务器端日志显示收到的是 GET

**原因：** nginx 中 `proxy_pass` 包含变量时（如 `proxy_pass http://$backend;`），nginx 会重新解析 DNS 并使用 HTTP/1.0，将 HEAD 自动转为 GET

**修复：** 改用 `stun.direct.conf` 中固定 `proxy_pass` 的 server 块（无变量）

#### 2. 多级子域名前缀丢失
**现象：** `1.fn.50001.stun.sd8.cc` 重定向到 `fn.bt.sd8.cc:13155` 而不是 `1.fn.bt.sd8.cc:13155`

**原因：** nginx `proxy_set_header Host $host` 会被正则处理，丢失多级前缀

**修复：** 改用 `proxy_set_header Host $http_host`

#### 3. 域名重定向完全失效
**现象：** 所有域名访问都返回管理后台页面

**原因：** `ROOT_DOMAIN` 环境变量为空

**修复：** 在管理后台 → 系统设置中配置服务域名

#### 4. sqlite3.Row.get() 报错
**现象：** 域名重定向时服务崩溃

**原因：** `sqlite3.Row` 对象没有 `.get()` 方法

**修复：** 改用 `row['column_name']`

## 六、域名重定向规则

### 域名格式
```
{前缀}.{端口}.你的域名
```

### 解析逻辑
1. 提取前缀和端口（如 `fn.50001.stun.sd8.cc` → 前缀=`fn`, 端口=`50001`）
2. 查找数据库中 `listen_port=50001` 的规则
3. 将规则的 `host` 中 `*` 替换为前缀
4. 重定向到 `{scheme}://{替换后的host}:{target_port}/`

### 示例

| 访问地址 | 规则 host | 重定向目标 |
|----------|-----------|------------|
| `fn.50001.stun.sd8.cc` | `*.bt.sd8.cc` | `http://fn.bt.sd8.cc:13155/` |
| `1.fn.50001.stun.sd8.cc` | `*.bt.sd8.cc` | `http://1.fn.bt.sd8.cc:13155/` |
| `a.a.fn.50001.stun.sd8.cc` | `*.bt.sd8.cc` | `http://a.a.fn.bt.sd8.cc:13155/` |

### 管理后台 "访问域名" 列格式

每条规则在管理后台的"访问域名"列按以下规则生成可点击链接（新标签页打开）：

| 来源 | 格式 | 颜色 |
|------|------|------|
| 主链接（domain_prefix） | `http://{prefix}.{listen_port}.{root_domain}/` | 绿色 |
| 端口直连 | `http://{root_domain}:{listen_port}/` | 灰色 |
| 域名映射 (domain_mappings) | 提取前缀后同上格式 `http://{extracted_prefix}.{listen_port}.{root_domain}/` | 蓝色 |

**domain_mappings 前缀提取规则：**
- `prefix domain` 格式（如 `s3 s3.bt.sd8.cc`）→ 用第一部分 `s3` 作为前缀
- 单域名格式（如 `4api.speedtest.cn.sd8.cc`）→ 提取第一个子域名 `4api` 作为前缀
- 全 URL 格式（如 `http://4api.speedtest.cn.sd8.cc:50001`）→ 先去协议/端口得到 `4api.speedtest.cn.sd8.cc`，再提取前缀

### 无 domain_prefix 的规则
如果有 `target_ip` 和 `target_port`，显示 `{scheme}://{target_ip}:{target_port}/` 作为灰色链接。

### 域名映射（domain_mappings）

`domain_mappings` 字段支持以下输入格式，用于指定自定义域名转发：

| 格式 | 示例 | 说明 |
|------|------|------|
| `prefix domain` | `sp sp.speedtest.cn.sd8.cc` | 显式指定前缀和后端域名 |
| 单域名 | `4api.speedtest.cn.sd8.cc` | 自动提取第一个子域名作为前缀 |
| 完整 URL | `http://4api.speedtest.cn.sd8.cc:50001` | 自动去除协议、端口、尾部斜杠 |

实际重定向逻辑中，会根据请求域名前缀匹配对应的 `domain_mappings` 条目，若匹配则使用映射的后端域名而非 `host` 模板。

### S3/MinIO 代理模式（proxy_mode=1）
- 创建规则时选择「代理模式」，nginx 直通目标服务器
- 客户端 Endpoint 设为 `http://s3.50002.stun.sd8.cc`（以规则配置为准）
- Host 头保留（S3 v4 签名必须）
- 所有 HTTP 方法（GET/POST/PUT/DELETE/HEAD）正常

#### S3 客户端配置示例
```
Endpoint: http://s3.50002.stun.sd8.cc
AccessKey: admin
SecretKey: w753159L
Region: us-east-1
```

## 八、数据库说明

### 数据库文件
- 路径：`/root/stun_redirect.db`
- 类型：SQLite
- 模式：WAL + synchronous=NORMAL + busy_timeout=5000

### 性能优化
SQLite 使用以下优化以支持高并发：

| 优化 | 说明 |
|------|------|
| WAL 模式 | 读写不互斥，读不阻塞写 |
| synchronous=NORMAL | 写入延迟降低 2-3x，不影响数据安全 |
| busy_timeout=5000ms | 写入冲突时自动等待重试，不报错 |
| 设置缓存 | `get_setting()` 结果永久缓存，写操作自动失效 |
| 规则缓存 | `get_rules_cache()` 每 3 秒刷新一次，`_domain_redirect` 不查 DB |

每个 S3 请求的 DB 读取次数：**0 次**（全走内存缓存）。

### 数据表

#### users（用户表）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| username | TEXT | 用户名（唯一） |
| password_hash | TEXT | 密码哈希 |
| role | TEXT | 角色：admin/user |
| created_at | TEXT | 创建时间 |

#### redirect_rules（规则表）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| fuwuqiportxuhao | TEXT | 端口序号（唯一） |
| listen_port | INTEGER | 监听端口（唯一） |
| target_ip | TEXT | 目标 IP |
| target_port | INTEGER | 目标端口 |
| host | TEXT | 目标主机模板（如 `*.bt.sd8.cc`） |
| redirect_scheme | TEXT | 协议：http/https |
| redirect_method | TEXT | 重定向方式：307/308 |
| cache_seconds | INTEGER | 缓存时间（秒） |
| created_by | TEXT | 创建者用户名 |
| enabled | INTEGER | 启用状态：0/1 |
| domain_prefix | TEXT | 域名前缀 |
| domain_mappings | TEXT | 域名映射（每行：前缀 域名） |
| proxy_mode | INTEGER | 0=重定向 1=代理 |
| created_at | TEXT | 创建时间 |
| updated_at | TEXT | 更新时间 |

#### settings（设置表）
| 字段 | 类型 | 说明 |
|------|------|------|
| key | TEXT | 设置键（主键） |
| value | TEXT | 设置值 |

默认设置：
| key | value | 说明 |
|-----|-------|------|
| root_domain | stun.sd8.cc | 服务域名 |
| admin_port | 8800 | 管理端口 |
| max_rules_per_user | 3 | 普通用户规则上限 |

#### login_attempts（登录尝试表）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| username | TEXT | 用户名 |
| ip | TEXT | IP 地址 |
| attempted_at | TEXT | 尝试时间 |

```bash
# 直接测试 Python 服务

# 通过 Nginx 测试
```


