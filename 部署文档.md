# STUN 代理/重定向服务 - 部署文档

## 项目简介

STUN 穿透管理服务，支持两种模式转发到实际 STUN 服务端口。

**功能特性：**
- Webhook 自动创建/更新规则
- 域名代理/重定向：`前缀.端口.stun.sd8.cc` → 实际服务地址
- 管理后台：用户管理、规则管理、系统设置
- RBAC 权限控制
- 防暴力破解
- iptables DNAT 端口转发（推荐给 S3/MinIO）

**默认登录：** admin / admin123

---

## 〇、转发模式说明

| 模式 | 描述 | 适用场景 | S3 兼容 |
|------|------|----------|---------|
| **重定向** (proxy_mode=0) | 返回 307/308，客户端重定向到目标 | 普通 HTTP 服务，如 frp | ❌ 签名必挂 |
| **代理模式** (proxy_mode=1) | nginx 直通后端或 Python 代理转发 | 任何服务，包括 S3/MinIO | ✅ 完美（Host 保留） |
| **iptables DNAT** | 内核级端口转发，零 HTTP 层 | S3/MinIO 高吞吐场景 | ✅ 完美（最推荐） |

### 如何选择
- **一般 HTTP 服务**（frp, web 等）：用**重定向模式**（性能最好，浏览器自动跟随）
- **S3/MinIO 等签名服务**：用 **iptables DNAT**（性能与重定向相同，但兼容所有协议）
- **不想开额外端口**：用**代理模式**（已可用，nginx 直通可达 100+ req/s）

## 一、服务端部署（Python）

### 1. 上传文件
将 `stun_redirect_server.py` 上传到服务器 `/root/` 目录

### 2. 安装依赖
无需安装，纯 Python 标准库

### 3. 启动服务
```bash
cd /root
setsid nohup python3 stun_redirect_server.py >> /tmp/stun.log 2>&1 &
```

### 4. 验证
```bash
# 检查进程
ps aux | grep stun_redirect_server

# 检查端口
netstat -tlnp | grep 8800

# 查看日志
tail -f /tmp/stun.log
```

### 5. 停止服务
```bash
pkill -f stun_redirect_server.py
```

## 二、宝塔 Nginx 配置

### 1. 添加站点
- 域名：`你的域名`（如 `stun.sd8.cc`）
- PHP 版本：纯静态

### 2. 添加反向代理
站点设置 → 反向代理 → 添加反向代理：
- 代理名称：`stun`
- 目标 URL：`http://127.0.0.1:8800`

### 3. 修改代理配置
**站点设置 → 配置文件**，找到 `location ^~ /` 块，修改为：

```nginx
location ^~ /
{
    proxy_pass http://127.0.0.1:8800;
    proxy_set_header Host $http_host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header REMOTE-HOST $remote_addr;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection $connection_upgrade;
    proxy_http_version 1.1;
    proxy_redirect off;
}
```

### 4. 修改 server_name
站点配置中 `server_name` 改为：

```nginx
server_name 你的域名 *.你的域名;
```

例如：
```nginx
server_name stun.sd8.cc *.stun.sd8.cc;
```

### 5. 重载 Nginx
```bash
nginx -t && nginx -s reload
```

## 三、关键配置说明

### 必须正确的配置项

| 配置项 | 正确值 | 错误值 | 影响 |
|--------|--------|--------|------|
| `proxy_set_header Host` | `$http_host` | `$host` | 多级子域名前缀丢失 |
| `proxy_pass` | `http://127.0.0.1:8800` | `http://外部IP:8800` | 请求绕行导致 Host 被改 |
| `proxy_redirect` | `off` | 默认 | Location 头被 nginx 改写 |
| `server_name` | `域名 *.域名` | 正则写法 | 多级子域名匹配异常 |

### 已知问题及修复

#### 1. 多级子域名前缀丢失
**现象：** `1.fn.50001.stun.sd8.cc` 重定向到 `fn.bt.sd8.cc:13155` 而不是 `1.fn.bt.sd8.cc:13155`

**原因：** nginx `proxy_set_header Host $host` 会被正则处理，丢失多级前缀

**修复：** 改用 `proxy_set_header Host $http_host`

#### 2. 域名重定向完全失效
**现象：** 所有域名访问都返回管理后台页面

**原因：** `ROOT_DOMAIN` 环境变量为空

**修复：** 在管理后台 → 系统设置中配置服务域名

#### 3. sqlite3.Row.get() 报错
**现象：** 域名重定向时服务崩溃

**原因：** `sqlite3.Row` 对象没有 `.get()` 方法

**修复：** 改用 `row['column_name']`

## 四、管理后台配置

### 系统设置（管理员）

登录后点击「系统设置」标签页：

| 设置项 | 说明 | 默认值 |
|--------|------|--------|
| 服务域名 | 根域名，用于解析重定向 | `stun.sd8.cc` |
| 普通用户规则上限 | 每个普通用户最多可创建的规则数 | `3` |

### 用户权限

| 功能 | 管理员 | 普通用户 |
|------|--------|----------|
| 查看所有规则 | ✅ | ❌ 只能看自己的 |
| 创建规则 | ✅ 无限制 | ✅ 受规则上限限制 |
| 编辑规则 | ✅ 所有 | ✅ 只能编辑自己的 |
| 删除规则 | ✅ 所有 | ✅ 只能删除自己的 |
| 用户管理 | ✅ | ❌ |
| 系统设置 | ✅ | ❌ |

### 防暴力破解

- 同一 IP + 用户名连续登录失败 **5 次**后锁定 **15 分钟**
- 锁定期间即使输入正确密码也无法登录
- 成功登录后清除该 IP + 用户名的失败记录

## 五、域名重定向规则

### 域名格式
```
{前缀}.{端口}.你的域名
```

### 解析逻辑
1. 提取前缀和端口（如 `fn.50001.stun.sd8.cc` → 前缀=`fn`, 端口=`50001`）
2. 查找数据库中 `listen_port=50001` 的规则
3. 将规则的 `host` 中 `*` 替换为前缀
4. 重定向到 `{scheme}://{替换后的host}:{target_port}/`

### 示例

| 访问地址 | 规则 host | 重定向目标 |
|----------|-----------|------------|
| `fn.50001.stun.sd8.cc` | `*.bt.sd8.cc` | `http://fn.bt.sd8.cc:13155/` |
| `1.fn.50001.stun.sd8.cc` | `*.bt.sd8.cc` | `http://1.fn.bt.sd8.cc:13155/` |
| `a.a.fn.50001.stun.sd8.cc` | `*.bt.sd8.cc` | `http://a.a.fn.bt.sd8.cc:13155/` |

## 六、iptables DNAT 端口转发（推荐 S3/MinIO）

iDNAT 在内核层面转发端口，客户端无感知、兼容所有协议。

### 1. 添加转发规则
```bash
# 格式：iptables -t nat -A PREROUTING -p tcp --dport {本地端口} -j DNAT --to-destination {目标IP}:{目标端口}

# 示例：59003 → 171.11.212.88:9239（MinIO S3）
iptables -t nat -A PREROUTING -p tcp --dport 59003 -j DNAT --to-destination 171.11.212.88:9239
iptables -A FORWARD -p tcp -d 171.11.212.88 --dport 9239 -m state --state NEW,ESTABLISHED,RELATED -j ACCEPT

# 启用 IP 转发（已启用可跳过）
echo "net.ipv4.ip_forward=1" >> /etc/sysctl.conf
sysctl -p
```

### 2. 客户端使用
```
# 访问方式
http://{服务器IP}:59003
# 或通过域名
http://59003.stun.sd8.cc  （如果配置了规则）

# 示例：MinIO S3 客户端配置
Endpoint: http://服务器IP:59003          # 直连，不经过 nginx
AccessKey: admin
SecretKey: w753159L
Region: us-east-1
```

### 3. 持久化规则
```bash
# 保存规则（Debian/Ubuntu）
apt install -y iptables-persistent
netfilter-persistent save

# 或 CentOS
yum install -y iptables-services
service iptables save
```

### 4. 常用管理命令
```bash
# 查看 DNAT 规则
iptables -t nat -L PREROUTING -n --line-numbers

# 删除规则（按行号）
iptables -t nat -D PREROUTING {行号}

# 清空所有端口转发
iptables -t nat -F
```

## 七、Webhook 配置

### GET 请求格式
```
http://服务器IP:8800/stun?ip=#{ip}&port=#{port}&fuwuqiportxuhao=#{fuwuqiportxuhao}&listen_port=#{listen_port}&user=#{user}
```

### 参数说明
| 参数 | 必填 | 说明 |
|------|------|------|
| `ip` | 是 | Lucky 获取的客户端 IP |
| `port` | 是 | Lucky 获取的客户端端口 |
| `fuwuqiportxuhao` | 否 | 服务器端口序号，不填则用 `listen_port` |
| `listen_port` | 否 | 监听端口，不填则自动分配 |
| `user` | 否 | 规则归属用户，默认 `admin1` |

#### redirect_rules（规则表）
| 字段 | 类型 | 说明 |

| fuwuqiportxuhao | TEXT | 端口序号（唯一） |
| listen_port | INTEGER | 监听端口（唯一） |
| target_ip | TEXT | 目标 IP |
| target_port | INTEGER | 目标端口 |
| host | TEXT | 目标主机模板（如 `*.bt.sd8.cc`） |
| redirect_scheme | TEXT | 协议：http/https |
| redirect_method | TEXT | 重定向方式：307/308 |
| cache_seconds | INTEGER | 缓存时间（秒） |
| created_by | TEXT | 创建者用户名 |
| enabled | INTEGER | 启用状态：0/1 |
| domain_prefix | TEXT | 域名前缀 |
| created_at | TEXT | 创建时间 |
| updated_at | TEXT | 更新时间 |

#### settings（设置表）
| 字段 | 类型 | 说明 |
|------|------|------|

#### login_attempts（登录尝试表）
| 字段 | 类型 | 说明 |
|------|------|------|

## 九、故障排查

### 检查服务状态
```bash
# 进程
ps aux | grep stun_redirect_server

# 端口
netstat -tlnp | grep 8800

# 日志
tail -20 /tmp/stun.log
```

### 检查 Nginx
```bash
# 配置测试
nginx -t

# 重载
nginx -s reload

# 查看代理配置
cat /www/server/panel/vhost/nginx/proxy/stun.sd8.cc/*.conf
```

### 测试重定向
```bash
# 直接测试 Python 服务
curl -v -H "Host: fn.50001.stun.sd8.cc" http://127.0.0.1:8800/

# 通过 Nginx 测试
curl -v -H "Host: fn.50001.stun.sd8.cc" http://127.0.0.1:80/
```

### 清除 Nginx 缓存
```bash
rm -rf /www/server/nginx/proxy_cache/*
nginx -s reload
```

### 清除登录锁定
```bash
sqlite3 /root/stun_redirect.db "DELETE FROM login_attempts;"
```

### 手动修改设置
```bash
# 修改服务域名
sqlite3 /root/stun_redirect.db "INSERT OR REPLACE INTO settings (key, value) VALUES ('root_domain', '你的域名');"

# 修改规则上限
sqlite3 /root/stun_redirect.db "INSERT OR REPLACE INTO settings (key, value) VALUES ('max_rules_per_user', '5');"
```
